# Pose Director

AI 拍照姿势模拟预览软件。应用会打开浏览器摄像头，用 MediaPipe 实时识别人体关键点，并根据内置姿势模板给出匹配度、引导框和自动拍摄倒计时。

## 功能

- 实时摄像头取景和人体姿态识别
- 基础姿势库：街头酷感、杂志站姿、松弛侧身
- 教程库：抱臂、正立、回头侧站、大佬感站姿、狠厉坐姿
- 基于关键动作、关节角度、身体朝向、手脚位置和 visibility 的姿势评分
- 按 fullBody / threeQuarter / halfBody / closeUp 做不同构图校验
- 仅“压迫感抱臂”和“冷静正立”允许自动倒计时拍摄，其余姿势只给评分和纠正建议
- 原片、自然美颜、增强人像三档高清导出
- 默认显示摄影取景引导，开发调试模式才显示完整骨架

## 本地运行

```bash
npm install
npm run dev
```

打开 Vite 输出的本地地址后，允许浏览器访问摄像头。姿态识别模型会从 MediaPipe 官方模型地址加载，首次运行需要网络连接。


## 项目结构

- `src/App.jsx`：主界面、拍摄流程、教程/姿势切换
- `src/services/poseDetection.js`：MediaPipe Pose / Face 按需加载和检测调度
- `src/services/poseScoring.js`：纯函数姿势评分、关键动作封顶和纠正建议
- `src/services/compositionCheck.js`：按镜头类型做入镜和画面位置校验
- `src/services/captureService.js`：按摄像头原始帧尺寸导出高清 JPEG
- `src/services/beautyService.js`：最终成片的轻量自然美颜处理
- `src/components/PoseCanvas.jsx`：实时骨架绘制
- `src/components/PoseGuideOverlay.jsx`：默认摄影引导覆盖层
- `src/data/poses.js` / `src/data/tutorials.js`：姿势模板和教程数据

## V2 判定规则

- 压迫感抱臂：双腕必须位于胸前或上腹区域，前臂跨过身体中线，双肘弯曲，身体接近正面，肩线基本水平；缺少抱臂动作时最高 55 分。
- 冷静正立：身体正对镜头，双肩和双髋基本水平，双臂自然下垂，双腿稳定站立；明显侧身、叉腰、抱手或大幅弯腿时最高 60 分。
- 街头酷感：必须有侧身或三分之二侧身、脚步错开、重心偏移和肩线控制；自然垂手直立不会超过自动拍照阈值。
- 杂志站姿：至少一只手位于腰侧或髋部附近，肩线放松且双腿稳定。
- 松弛侧身：身体侧转，至少一只手靠近脸侧，双脚需要有前后或左右层次。
- 侧身插袋回头：身体侧向，手在腰侧/口袋附近，双脚错开。
- 大佬感站姿：正面或轻微正面，肩膀打开，腿部稳定并有重心张力。
- 狠厉坐姿：半身构图，脸肩臂腰髋可见，手臂需要形成支点感。

## 自动拍摄

自动拍摄只对以下姿势开启：

- 压迫感抱臂：真实匹配度不低于 72，构图合格，关键动作合格，并稳定 1.5 秒。
- 冷静正立：真实匹配度不低于 70，构图合格，关键动作合格，并稳定 1.2 秒。

倒计时期间只要分数低于阈值、构图失败、关键动作失败、人体离开画面或切换模板，倒计时会立即取消。其他姿势保留手动拍摄，但绝不自动拍照。

## 手机验收

1. 用手机浏览器打开本地或 Vercel 地址，允许摄像头权限。
2. 选择“街头酷感”，自然垂手站立应低于 80 分，且不会自动拍照。
3. 选择“压迫感抱臂”，不抱臂时应封顶在 55 分左右，并提示双手抬到胸前交叉抱臂。
4. 正确抱臂或冷静正立时，保持构图和动作稳定才会进入倒计时。
5. 拍摄完成后检查结果页的图片尺寸和文件大小，保存的应是摄像头原始帧尺寸导出的高清 JPEG。

模型文件不需要手动下载，运行时会从 MediaPipe 官方地址加载；首次打开需要网络连接。
